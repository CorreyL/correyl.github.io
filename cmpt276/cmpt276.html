In this course I was tasked with building a back-end for a hypothetical web-based or mobile application (such as a social network), specifically to store user-inputted data such as usernames, passwords, status updates, etc.

The back-end was set up as a tiered service as 4 different servers:

1. Low-Level Interface: Database Layer
The Database Layer was an interface to the Storage Layer, specifically Microsoft Azure. This ensured that if the application needed to be moved to another Cloud Storage Service, such as Amazon DynamoDB or Google Cloud Datastore, the only code that needed to be changed would be in this lowest-level service, without having to change any other part of the appllication.
